## Summary

**Content Identification Method:**
- Generate cache_key from query + source_ids + price
- Hash with SHA256, take first 32 characters
- Use cache_key to lookup existing content_id
- Store mapping permanently for future reuse

**Key Insight:**
We identify content by WHAT was analyzed (query + sources), not by the result. This allows us to:
1. Check if content exists BEFORE generating it
2. Reuse content_ids across users and time
3. Detect "already purchased" accurately
4. Prevent duplicate registrations with LedeWire
